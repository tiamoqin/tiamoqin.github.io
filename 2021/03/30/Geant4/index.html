Geant 4.10.7 的编译与安装

欧洲核子研究组织（European organization for nuclear research, CERN）开发的开源几何径迹模拟软件Geant4（CERN, 欧洲），能够模拟包含电子、质子，重离子等各种粒子在物质中的输运过程。用于DNA的放射生物学蒙特卡罗模拟软件Geant4-DNA（National Institute for Nuclear and Particle Physics：IN2P3，法国）是几何径迹模拟软件Geant4的扩展包，可以研究电离辐射在DNA上引起的早期生物损伤，同样，Geant-DNA也是开源的。作为第三个主要的在微米和纳米几何级别研究电离辐射导致的生物损伤研究平台。

作为之前的主要模拟工具，Geant4 陪伴了我四年。2020年12月更新了Geant4.10.7版本。与之前的版本相比，改动还是挺多的。譬如G4RunManager类被G4TaskRunManager和 G4RunManagerFactory替代，对于DNA，添加了IRT化学模块。具体可以参考说明Geant4.10.7 Release Notes。

本安装仅限于基于Ubuntu 16.04的安装，并且开启CLion IDE调试。

安装注意事项：在安装之前，必须卸载所有Geant 4.10.7之前的版本。否则make时会出现时别不了一些符号的问题。原因时：If there was an existing build of the application in that directory, reconfiguring may not have worked fully. CMake caches information when it runs so it doesn’t need to do expensive checks every time you build. So in this case it might have rebuilt against the old version, then tried to run against the new (via LD_LIBRARY_PATH).（懒得翻译了）类似安装故障.如果安装过程中出现其他问题，可以在Geant4 forum查找相似情况或直接询问。

Step 1: Geant4 System/Software Prerequisites

  • Geant4 Toolkit Source Code.

  • C++ Compiler and Standard Library supporting the C++11 Standard:

    • Linux: GNU Compiler Collection 4.9.3 or higher.

  • CMake 3.8 or higher.

    需要可视化驱动的

  • Qt5 User Interface and Visualization (All Platforms)

    • Qt5 headers and libraries
    • OpenGL or MesaGL headers and libraries.

    其他根据需要，参考安装文档

Step 2: Download Packages

首先进入官网(http://geant4.web.cern.ch/support/download)下载源代码（推荐tar.gz格式）及数据文件，解压。新建一个文件夹专门用来放`Geant4`相关文件，新建data，source，build文件夹，将Geant4的文件复制进来并按如下结构组织：

.
├── build  
├── data
│   ├── G4ABLA3.1
│   ├── G4EMLOW7.13
│   ├── G4ENSDFSTATE2.3
│   ├── G4INCL1.0
│   ├── G4NDL4.6
│   ├── G4PARTICLEXS3.1.1
│   ├── G4PII1.3
│   ├── G4SAIDDATA2.0
│   ├── G4TENDL1.3.2
│   ├── PhotonEvaporation5.7
│   ├── RadioactiveDecay5.6
│   └── RealSurface2.2
└── source
    └── geant4.10.07.p01

安装也可以参考官方发布的安装文档。上面的data需要提前从官网下载源代码，然后解压到data文件夹中。

Step 3： Install Dependencies

安装编译所需环境：

sudo apt-get install build-essential cmake

安装相关依赖：

sudo apt-get install libgl1-mesa-dev libglu1-mesa-dev libxt-dev libxmu-dev libxi-dev zlib1g-dev libgl2ps-dev libexpat1-dev libxerces-c-dev

Step 4 : Compile

进入build文件夹，用cmake命令：

cmake ../source/geant4.10.07.p01/ -DCMAKE_BUILD_TYPE=DEBUG -DGEANT4_USE_GDML=ON -DGEANT4_USE_OPENGL_X11=ON -DGEANT4_USE_RAYTRACER_X11=ON -DGEANT4_BUILD_MULTITHREADED=ON

其中../source/geant4.10.07.p01/替换成（如果版本不一样）你自己的Geant4源代码所在目录，需要QT则加上-DGEANT4_USE_QT=ON。如果不需要调试则把-DCMAKE_BUILD_TYPE=DEBUG改成-DCMAKE_BUILD_TYPE=RELEASE。-DGEANT4_BUILD_MULTITHREADED=ON是多线程，视情况开启。

完成之后开始编译：

make -jX

-jX为多线程编译，如-j8。

编译完成之后进行安装：

sudo make install

Step 5: Configure

安装的默认路径在/usr/local/share/Geant4-10.7.1，将下载的数据文件复制到该文件夹：（注意，如果以前装过其他版本，在这里都要删除）

sudo cp -r ./data/ /usr/local/share/Geant4-10.7.1/

之后，在gedit ~/.bashrc里添加/usr/local/share/Geant4-10.7.1/geant4make/geant4make.sh，如果你的版本和我的不一样，相应修改即可。添加后 source ~/.bashrc生效。

Step 6: CLion Configuration

最后我们来配置CLion环境，配好之后可以在IDE中编写Geant4代码，还可以断点调试，非常方便。安装CLion的过程这里省略，打开一个Geant4自带的例子或者自己新建一个项目，打开Edit Configurations。

随便打开打开一个终端，输入一下命令获取环境变量：

env | grep G4

在Environment variables填入刚才获取的环境变量（复制之后按一下粘贴就可以了），然后把Working directory设置成当前文件夹。

其他可能出现的问题：

cmake中找不到Qt，首先在gedit ~/.bashrc里添加

export QTDIR=/opt/Qt/5.12.2/gcc_64 export LD_LIBRARY=$QTDIR/lib export PATH=$QTDIR/bin:$QTDIR/../Tools/QtCreator/bin:$PATH export LD_LIBRARY_PATH=$QTDIR/lib:$LD_LIBRARY_PATH
